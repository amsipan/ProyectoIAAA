Síntoma actual (reporte Bryan tras commit 1454216):
  Ya NO aparece la cruz (crosshair). Pero SÍ aparece un cursor de flecha/mouse
  normal, "medio transparente", encima de la tijera dibujada. Bryan quiere que
  el ÚNICO puntero visible sea la tijera (glyph canvas), como TradingView.

Comportamiento esperado: cursor del SO completamente invisible sobre price/atr
canvas (y frames del plot) mientras _replay_select_mode == 1.

Implementación actual (_select_mode_blank_cursor en ChartEngine.pm):
  1. configure(-cursor => '')  — cadena vacía
  2. fallback: 'left_ptr' si probe falla

Por qué se eligió '':
  Probe runtime en Fedora35/WSLg (perl Tk headless, misma distro):

    none     → FAIL  bad cursor spec "none"
    blank    → FAIL  bad cursor spec "blank"
    @/tmp/blank_cursor.xbm → FAIL  bad cursor spec (sintaxis @file no soportada)
    Bitmap inline → FAIL  format error in bitmap data
    Bitmap como -cursor => $bmp → FAIL  bad cursor spec "imageN"
    '' (vacío) → OK, cget devuelve cadena vacía

  Con '' el test headless dice éxito; Bryan en WSLg GUI ve flecha semitransparente.
  Hipótesis: WSLg/XWayland no oculta el puntero con cursor Tk vacío; muestra
  fallback del compositor (ghost arrow) o hereda cursor del ancestro ($mw / frame).

Intentos previos (commits ae3326f, a28d714) también fallaron:
  - assets/blank_cursor.xbm (1x1)
  - none / blank
  - suprimir crosshair dibujado (eso sí funciona)
